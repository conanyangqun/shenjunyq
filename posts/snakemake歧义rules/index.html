snakemake歧义rules

最近在用snakemake改写一个PB HiFi数据的分析流程，遇到了snakemake的歧义rules的问题。被困扰了许久后，最终发现原来是一个没注意的小问题导致的。

歧义rules

歧义rules（ambiguous rules）指两个rules可以产生相同的输出文件，snakemake无法决定使用哪个，从而报AmbiguousRuleException错误。

例如以下示例：

rule all:
  input:
    [
      f"results/hg002/dv/hg002.dv.{suffix}"
      for suffix in ['vcf.gz', 'vcf.gz.tbi']
    ]

rule expected:
  output:
    vcf = "results/{sample}/dv/{sample}.dv.vcf.gz",
    vcf_index = "results/{sample}/dv/{sample}.dv.vcf.gz.tbi"
  shell:
    "hostname"

rule bgzip_vcf:
  input: "results/{prefix}.vcf"
  output: "results/{prefix}.vcf.gz"
  shell:
    "bgzip {input}"

rule tabix_vcf:
  input: "results/{prefix}.vcf.gz"
  output: "results/{prefix}.vcf.gz.tbi"
  shell:
    "tabix -p vcf {input}"

expected这个预期的rule执行一定的分析任务，生成VCF和对应的index文件。同时为了满足可以压缩并索引任意的VCF文件，编写了bgzip_vcf, tabix_vcf两个rules。注意，这两个rule涵盖的范围比较广，并不符合snakemake的建议。应该尽量避免。

用snakemake --cores -n -p可以进行测试。

如果只指定xxx.vcf.gz作为输出文件，那么分析流程会按照预期进行。这是因为虽然bgzip_vcf也输出了同样的文件，但是并不存在它的输入文件，因此这个rule无法执行。所以只有expected满足执行条件。

但是如果指定xxx.vcf.gz.tbi作为输出文件，则会报出tabix_vcf, expected歧义错误。因为两个rule的执行条件都能满足。

解决这个问题的办法，要么使用ruleorder指定执行顺序，要么避免写tabix_vcf的这种覆盖范围比较大的rule。另外，也可以指定rule之间的依赖关系。

rule执行判定

我仿照的流程中也使用了类似的覆盖范围比较大的rule，但是仿照的流程可以正常运行。于是我决定探究一下导致这个问题的原因。

仿照的流程之所以成功运行，假设是因为tabix_vcf这条rule不满足执行条件，之所以不满足是因为bgzip_vcf这条rule不满足执行条件。但是这个假设本身就是错误的，因为除非明确指定依赖，否则rule之间是独立的。

在这个错误的假设下，我debug了很久，始终没有找到导致问题的原因。于是我抽离了下面这个例子来测试：

rule all:
  input:
    [
      f"results/hg002/dv/hg002.dv.{suffix}"
      for suffix in ['vcf.gz', 'vcf.gz.tbi']
    ]

rule expected:
  output:
    vcf = "results/hg002/dv/hg002.dv.vcf.gz"
    vcf_index = "results/hg002/dv/hg002.dv.vcf.gz.tbi"
  shell:
    "hostname"

rule bgzip_vcf:
  input: "results/{prefix}.vcf"
  output: "results/{prefix}.vcf.gz"
  shell:
    "bgzip {input}"

rule tabix_vcf:
  input: "results/{prefix}.vcf.gz"
  output: "results/{prefix}.vcf.gz.tbi"
  shell:
    "tabix -p vcf {input}"

snakemake --cores -n -p可以成功运行。对比两个例子可以发现，expected的输出文件存在不同，成功运行的这个流程中并没有使用通配符。所以，我推测对于歧义的rule，不含通配符的rule的优先级会高于带通配符的rule的优先级。在这个例子中，expected的优先级高于tabix_vcf。而我之前的假设，是完全错误的。

后记

在经历了长时间的失败之后，对心态的影响是明显的，尤其是在自己的意识中这个问题并不复杂时。但是造成这种状态的原因，完全是自己的盲目尝试。

在进行尝试之前，需要先根据已有的知识设立假设，然后设计实验去验证这个假设。最初我设立的假设，本身就是错误的（压根都没仔细考虑导致的原因）。但是目前对snakemake还不是特别的熟悉，并没有发现这个假设的错误性。

条条大路通罗马。在遇到这个bug时，完全可以借助其他的方法绕过去。而我选择探究一下，是因为时间尚充足。
